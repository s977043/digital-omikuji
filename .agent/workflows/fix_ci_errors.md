# CIエラー修正フロー

CI（Lint, Type Check, Test）のエラーをローカルで修正・確認するためのワークフローです。
コードを変更した後や、PR提出前に実行することを推奨します。

## 1. Lintエラーの自動修正

PrettierやESLintのフォーマットエラーを可能な限り自動修正します。

// turbo
```bash
pnpm lint:fix
```

## 2. 型チェックの実行

TypeScriptの型エラーがないか確認します。
エラーが出力された場合は、内容を確認して手動で修正してください。
（ヒント: `tsconfig.json` の設定や、外部ライブラリの型定義の確認が必要になる場合があります）

// turbo
```bash
pnpm tsc --noEmit
```

## 3. テストの実行

ユニットテストを実行し、変更によって既存の機能が壊れていないか確認します。

// turbo
```bash
pnpm test
```

## 4. 次のアクション

- 全てのコマンドが成功した場合: コミットしてプッシュしてください。
- エラーが残っている場合: エラーログに記載されたファイルを確認し、修正を行ってから再度このワークフローを実行してください。
