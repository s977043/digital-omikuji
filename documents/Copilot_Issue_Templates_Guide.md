# Copilot Issue Templates Usage Guide

## 概要

このリポジトリには、GitHub Copilot や AI アシスタントにタスクをアサインする際に最適化されたIssueテンプレートが用意されています。これらのテンプレートは、AIが効率的にタスクを理解し、実行できるよう構造化された情報を提供します。

## 利用可能なテンプレート

### 1. 機能開発テンプレート (`copilot-feature-request.md`)
**用途:** 新機能の開発をCopilotにアサインする場合
- 機能要件と非機能要件の詳細定義
- 技術仕様と実装方針
- 段階的な実装計画
- テスト・検証項目

### 2. バグ修正テンプレート (`copilot-bug-fix.md`)
**用途:** バグの修正をCopilotにアサインする場合
- 詳細な再現手順と環境情報
- 影響範囲の分析
- 修正手順とテスト計画
- 再発防止策

### 3. コンポーネント開発テンプレート (`copilot-component-development.md`)
**用途:** 新しいReactコンポーネントの開発をアサインする場合
- UI/UX要件とProps設計
- アクセシビリティ要件
- 実装ガイドライン
- 品質保証項目

### 4. リファクタリングテンプレート (`copilot-refactoring.md`)
**用途:** コードのリファクタリングをアサインする場合
- 現状分析と改善目標
- 段階的なリファクタリング計画
- 品質メトリクスの測定
- 改善効果の評価

### 5. ドキュメントテンプレート (`copilot-documentation.md`)
**用途:** ドキュメントの作成・更新をアサインする場合
- ドキュメント要件と対象読者
- コンテンツ構成と品質要件
- 作成プロセスと品質保証
- 保守・更新計画

### 6. 一般タスクテンプレート (`copilot-general-task.md`)
**用途:** 上記以外の一般的な開発タスクをアサインする場合
- 柔軟なタスク定義
- 段階的な実行計画
- 品質保証とテスト
- 成果物と引き渡し

## テンプレートの使用方法

### 1. 新しいIssueの作成
1. GitHubリポジトリで「New Issue」をクリック
2. 適切なCopilotテンプレートを選択
3. テンプレートに従って必要な情報を入力
4. `copilot-assigned` ラベルと`@copilot` をアサインに追加

### 2. テンプレートの活用ポイント
- **[]** で囲まれた箇所は具体的な内容に置き換える
- チェックボックス項目は実行状況の追跡に使用
- 各セクションの役割分担を明確にして記入
- 技術制約や品質要件を具体的に記載

### 3. 既存システムとの連携
テンプレートは既存のGemini CLIシステムと連携しています：

```bash
# 利用可能なコマンド
npm run gemini:component    # コンポーネント生成
npm run gemini:review       # コードレビュー
npm run gemini:check-docs   # ドキュメント確認
npm run dev                 # 開発サーバー
npm run test               # テスト実行
```

## テンプレート設計の特徴

### 1. 構造化された情報
- 役割別セクション（要件定義者、アーキテクト、実装者、品質保証）
- 段階的な作業フロー
- 明確な完了条件

### 2. AI最適化
- 曖昧性を排除した明確な指示
- 技術的制約の明示
- 参考リソースとツールの提供

### 3. 品質保証
- 包括的なテスト項目
- レビュー観点の明示
- ドキュメント更新要件

### 4. プロジェクト一貫性
- 既存コーディング規約との統合
- Gemini CLIとの連携
- 日本語での詳細な説明

## 注意事項

1. **セキュリティ:** 機密情報はIssueに記載しないよう注意
2. **更新:** プロジェクトの進化に合わせてテンプレートも更新
3. **フィードバック:** 使用結果をもとにテンプレートの改善を継続
4. **品質:** AIアサインでも人間によるレビューは必須
