# Digital Omikuji コードレビュースタイルガイド

このドキュメントは Gemini Code Assist がコードレビューを行う際のガイドラインです。

## プロジェクト概要

- **アプリ名**: Digital Omikuji (2026年 新年おみくじアプリ)
- **フレームワーク**: Expo SDK 52 (React Native)
- **ルーティング**: Expo Router v4
- **スタイリング**: NativeWind v4 (Tailwind CSS)
- **アニメーション**: Moti (Reanimated)
- **言語**: TypeScript (Strict Mode)

## コーディング規約

### TypeScript

- `any` 型の使用を避ける。型定義を明確にする。
- インターフェースを使用してプロップスと状態を定義する。
- `async/await` を使用して非同期処理を行う。

### React / React Native

- 関数コンポーネントと Hooks を使用する。
- `useCallback` / `useMemo` を適切に使用してパフォーマンスを最適化する。
- 状態更新関数には更新関数形式 `setState(prev => ...)` を推奨する。

### スタイリング

- NativeWind の `className` プロップを使用する。
- インラインスタイルは最小限に抑える。

### エラーハンドリング

- 外部 API 呼び出しには `try/catch` を必須とする。
- ユーザーに適切なフィードバックを提供する。

## レビュー時の注意点

### 重視すべき項目

1. **型安全性**: 型定義が適切か
2. **パフォーマンス**: 不要な再レンダリングがないか
3. **エラーハンドリング**: 例外処理が適切か
4. **セキュリティ**: 機密情報の露出がないか

### 軽微な指摘として扱う項目

1. コメントの追加提案
2. 変数名の軽微な改善提案
3. フォーマットの細かい調整

## 禁止事項

- `.env*` ファイルへの変更
- `node_modules` への変更
- ロックファイル (`*.lock`) の手動編集
