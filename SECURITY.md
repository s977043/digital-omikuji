## 脆弱性の報告

セキュリティ上の脆弱性を発見した場合は、公開的な Issue ではなく、以下の方法で報告してください：

### 推奨される報告方法

1. **GitHub Security Advisories**（推奨）
   - リポジトリの "Security" タブから "Report a vulnerability" を使用してください
   - プライベートに報告でき、修正まで非公開で管理されます

2. **メンテナーへの直接連絡**
   - Issue やプルリクエストではセキュリティ問題を報告しないでください
   - 公開される前に修正する機会を提供するため、プライベートな報告をお願いします

### 報告に含めるべき情報

- 脆弱性の詳細な説明
- 再現手順
- 潜在的な影響の評価
- 修正案（もしあれば）

### 対応プロセス

1. **確認**: 24-48時間以内に報告の受領を確認します
2. **調査**: セキュリティチームが脆弱性を調査・評価します
3. **修正**: 必要に応じて修正を実装します
4. **公開**: 修正後、適切なタイミングで脆弱性を公開します

## セキュリティのベストプラクティス

このプロジェクトを使用する際は：

- 最新バージョンを使用してください
- 依存関係を定期的に更新してください
- 本番環境では適切な環境変数設定を行ってください
- `.env` ファイルを公開リポジトリにコミットしないでください
