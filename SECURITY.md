# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |

## Reporting a Vulnerability

We take the security of Digital Omikuji seriously.
If you discover a security vulnerability, please report it responsibly.

### How to Report

1. **GitHub Private Security Advisory** (Recommended):
   - Navigate to [Security Advisories](https://github.com/s977043/digital-omikuji/security/advisories/new)
   - Click "Report a vulnerability"
   - Provide detailed information about the vulnerability

2. **Email**:
   - Contact the maintainers directly through GitHub

### What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### Response Timeline

- **Initial Response**: Within 48 hours
- **Status Update**: Within 7 days
- **Resolution Target**: Within 30 days (depending on severity)

### Disclosure Policy

- Please do not disclose the vulnerability publicly until we have addressed it
- We will acknowledge your contribution in the release notes (unless you prefer anonymity)
- We aim to resolve critical vulnerabilities as quickly as possible

## Security Best Practices

This project follows security best practices including:

- Regular dependency updates via Dependabot
- No storage of sensitive user data
- Environment variables for sensitive configuration
- Sentry for error monitoring (without PII collection)

## Scope

This policy applies to the Digital Omikuji mobile application and its related repositories.
